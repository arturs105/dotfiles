# Commit

Create a git commit for the current changes.

## Process

1. Run `git status` (never use `-uall`) and `git diff` + `git diff --cached` to understand all changes.

2. Stage all relevant changed files. Prefer adding files by name rather than `git add -A`. Never stage files that likely contain secrets (.env, credentials, etc.) — warn the user instead.

3. Compose a commit message:
   - **Subject line**: Single emoji + uppercase concise summary. Max ~70 chars. Sacrifice grammar for brevity.
   - **Body** (optional): If the change is non-trivial, add a blank line then a short explanation of *why*. Keep it brief.
   - Pick the emoji freely — whatever best fits (e.g. ✨ new feature, 🐛 bug fix, ♻️ refactor, 🔧 config/tooling, ✅ tests, 📝 docs, 🎨 formatting, ⚡ performance, 🔥 remove code, 🚚 move/rename, 💄 UI, 🏗️ architecture). Not limited to this list.
   - No co-author lines, AI attribution, or "generated by" mentions. Ever.

4. If `$ARGUMENTS` is provided, use it as guidance for the message content. Otherwise infer from the diff.

5. Commit using a HEREDOC:
   ```bash
   git commit -m "$(cat <<'EOF'
   <emoji> Subject line

   Optional body
   EOF
   )"
   ```

6. Run `git status` after to verify success. Show the user the commit hash and message.
